
## 📁 FileFlux - 简易文件传输服务器

### 🚀 项目简介

**FileFlux** 是一个基于 Python 的轻量级文件传输服务器，旨在简化文件的上传、下载和共享过程。它特别适用于以下场景：

- **开发环境文件传输**：在工作环境中，特别是在需要通过跳板机连接到开发机时，FileFlux 可以方便地进行文件的上传和下载。
- **云服务器文件管理**：在使用阿里云、百度云等云服务器进行开发时，可以通过 FileFlux 实现文件的便捷管理。
- **大文件分享**：对于微信、QQ 等平台无法直接传输的大文件，可以通过 FileFlux 生成下载链接进行分享。


### 📋 主要功能

- **🚀 极速并发支持**：后端升级为多线程服务器 (ThreadingHTTPServer)，支持多人同时上传下载，互不阻塞，大幅提升多人协作效率。
- **☁️ 拖拽与批量上传**：新增现代化拖拽上传区域，支持一次性拖入或选择多个文件进行批量上传。
- **🔒 安全性增强**：
	强制下载防护：所有文件下载强制设置为 application/octet-stream 附件形式，防止浏览器直接执行恶意代码（如 HTML/SVG XSS 攻击）。
	路径遍历防护：增强后端路径检查逻辑，严防通过 ../ 等方式访问服务器其他敏感路径。
- **📊 实时进度条**：上传过程中显示精确的进度百分比与可视化进度条，大文件上传状态一目了然。
- **📱 响应式 UI 设计**：界面全面升级，适配移动端与桌面端，提供更现代、整洁的视觉体验。
- **📂 基础功能保留**：继续支持中文文件名上传/下载、文件列表自动刷新、文件删除及详情查看。


### 📥 安装与部署

#### 1. **克隆项目**

```bash
git clone <your-repo-url>
cd FileFlux
```

#### 2. **安装依赖**

本项目仅依赖 Python3 标准库，无需额外安装第三方包。

#### 3. **配置项目**

- **配置文件**：项目根文件夹下的 `config.json` 文件用于配置服务器端口和上传路径。

  示例配置：
  ```json
  {
      "port": 8000,
      "upload_dir": "uploads"
  }
  ```


#### 4. **启动服务**

在项目根文件夹下运行以下命令启动服务器：

```bash
python3 server.py
```

默认情况下，服务器将在 `http://localhost:8000` 上运行。


### 🖥️ 使用说明

#### 1. **访问主页**

打开浏览器，访问 `http://localhost:8000`，即可看到 FileFlux 的主界面。

#### 2. **上传文件**

- 点击“选择文件”按钮，选择要上传的文件。
- 点击“Upload”按钮，开始上传文件。
- 上传成功后，页面会显示“上传成功”提示，并在文件列表中展示该文件。

#### 3. **下载文件**

- 在文件列表中找到要下载的文件，点击文件名即可开始下载。



### 🆕  版本记录

### ✅ v1.4 大文件分片上传与进度优化（最新更新）

- **分片上传重构**
  前端将大文件自动切分为 50MB 的分片，逐片通过 XHR 发送至后端 `/upload_chunk` 接口，每个分片使用标准 Content-Length，彻底解决大文件（5GB+）上传时浏览器内存溢出卡死的问题，同时避免了 chunked transfer encoding 的兼容性问题。

- **后端分片接收**
  新增 `/upload_chunk` POST 端点，通过请求头传递文件名、分片索引、总分片数等元数据，首片创建文件、后续追加写入，读取缓冲区从 8KB 提升至 64KB。

- **详细上传进度**
  进度条下方新增实时详情行，展示已上传/总大小、当前分片编号、实时传输速度、预计剩余时间等信息，大文件上传状态一目了然。


### ✅ v1.3 架构升级与安全强化

- **后端架构升级**  
  从单线程 HTTPServer 迁移至 ThreadingHTTPServer，解决多用户阻塞问题。

- **安全加固**  
  移除 MIME 类型猜测，强制流式下载以防御 XSS；增加 safe_join 逻辑防止路径遍历攻击。

- **前端交互优化**  
  引入拖拽上传 API，支持批量文件处理；重写 CSS 样式表，提供更加现代化的视觉效果。

- **预览模式稳定性**  
  增加了前端稳定机制，在后端未连接时自动展示演示数据，方便 UI 调试。


### ✅ v1.2 新增功能

- **上传进度条功能**  
  在上传过程中动态显示进度条，提升用户对上传状态的感知，优化大文件上传体验。

- **中文文件名完整支持**  
  前端与后端协同优化，实现上传、展示、下载全过程支持中文文件名，避免 URL 编码问题。

- **文件名解码展示**  
  文件列表中显示原始中文文件名，提升可读性，实际下载仍使用编码后的安全格式。


#### ✅ v1.1 新增功能

- **文件删除功能**  
  支持通过 Web 界面删除服务器上的文件。每个文件项右侧提供“Delete”按钮，点击后可弹窗确认并从服务器移除文件。

- **文件详情查看功能**  
  点击文件名会弹出模态框，展示以下信息：
  - 文件名
  - 文件大小（字节）
  - 最后修改时间  
    同时提供一键下载入口，确保用户体验流畅。

- **增强交互体验**  
  使用模态框（Modal）替代原生 `alert()`，提升视觉效果与操作友好性，同时保留原有界面风格不变。
